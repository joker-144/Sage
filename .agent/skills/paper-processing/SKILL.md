# Paper Processing Skill

学术论文文档处理技能包 — Sage 系统的文档解析与预处理核心。

## 1. 角色定位

本技能是 Sage 论文写作系统的"文档输入入口"，负责将用户上传或加载的论文（PDF/Word/LaTeX/扫描版）转换为可被索引、检索、引用的纯文本结构。所有后续的文献检索、引用插入、查重都依赖于本技能产出的高质量文本。

## 2. 核心能力

| 能力 | 工具 | 适用场景 |
|------|------|----------|
| PDF 解析 | `parse_pdf` | 标准 PDF 论文 |
| Word 解析 | `parse_docx` | .docx 格式论文/草稿 |
| LaTeX 解析 | `parse_latex` | arXiv 源码/期刊投稿源 |
| 元数据提取 | `extract_metadata` | 标准化论文信息 |
| OCR 识别 | `ocr_document` | 扫描版/图片型文档 |

## 3. 详细执行手册

### 3.1 文档解析流程

#### 步骤一：识别文档格式
- 根据文件扩展名（.pdf/.docx/.tex）选择对应解析器
- 若扩展名缺失或可疑，先尝试 `read_file` 读取头部字节判断

#### 步骤二：调用对应解析工具
- **PDF** → `parse_pdf`（依赖 PyMuPDF/fitz）
- **Word** → `parse_docx`（依赖 python-docx）
- **LaTeX** → `parse_latex`（纯文本解析，保留 \section/\subsection 结构）

#### 步骤三：验证解析质量
解析完成后必须验证：
- [ ] 文本是否完整（无大面积空缺）
- [ ] 是否存在乱码（编码问题）
- [ ] 章节结构是否保留（标题层级）
- [ ] 公式/图表是否被合理处理（至少有占位说明）

若解析质量不达标：
- PDF 出现大面积空白 → 怀疑扫描版，转用 `ocr_document`
- Word 出现段落丢失 → 检查是否有文本框内容未提取
- LaTeX 出现编译错误 → 提示用户提供完整源码包

### 3.2 元数据提取规范

调用 `extract_metadata` 时应输出结构化信息：

```
- 标题 (title): 论文完整标题
- 作者 (authors): 作者列表，按贡献顺序
- 年份 (year): 出版年份
- DOI: 数字对象标识符（若有）
- 摘要 (abstract): 论文摘要原文
- 关键词 (keywords): 作者关键词
- 期刊/会议 (venue): 发表场所
- 学科领域 (domain): 自动识别（SCI/SSCI/CSSCI/EI 分区）
```

**识别策略**：
1. 优先从文档前部区域提取（标题/作者/摘要通常在首2页）
2. DOI 优先匹配模式：`10.\d{4,}/\S+`
3. 若作者格式不规范（如含机构信息），分离个人名与机构
4. 学科领域识别基于关键词与期刊名匹配

### 3.3 OCR 处理特殊规则

`ocr_document` 仅在以下情况调用：
- `parse_pdf` 返回文本为空或字数过少（< 100字/页）
- 文档被识别为扫描版（图片型PDF）
- 用户明确指定为扫描件

**注意事项**：
- OCR 耗时显著高于文本提取，非必要不调用
- OCR 结果可能存在识别错误，应在引用前人工/智能体复核
- 学术公式、表格、图表 OCR 识别率低，应保留原文占位符

### 3.4 LaTeX 解析特殊规则

`parse_latex` 处理 LaTeX 源文件时：
- 保留 `\section`/`\subsection`/`\subsubsection` 作为章节边界
- 移除 `\cite{...}`、`\ref{...}`、`\label{...}` 的 LaTeX 命令但保留引用键
- 数学环境（`$...$`/`\[...\]`/`equation`）转换为可读形式
- 注释（`%` 开头）默认移除，但保留有意义的 TODO/note

## 4. 与其他技能的协作

| 下游技能 | 协作方式 |
|---------|---------|
| `literature-index` | 解析后的文本交付给索引工具建立向量索引 |
| `writing-assistant` | 提取的元数据用于生成参考文献条目 |
| `external-search` | 提取的 DOI/标题用于外部检索查证 |

## 5. 质量标准

- 文本提取完整度 ≥ 95%（论文正文）
- 元数据字段完整度 ≥ 80%（至少含标题、作者、年份）
- 章节结构保留率 100%（标题层级不丢失）
- 解析耗时：单篇 PDF < 5秒，OCR 模式 < 30秒

## 6. 智能体自主判断事项

本技能包仅提供执行流程与质量标准，以下决策由智能体根据上下文自主判断：
- 是否需要 OCR（基于首次解析结果质量）
- 是否需要二次解析（首次结果不达标时）
- 解析结果的后续使用方式（仅索引 / 同时引用 / 仅元数据）
- 是否调用 `extract_metadata`（短文档可跳过）

## 7. 依赖说明

| 依赖 | 用途 | 安装命令 |
|------|------|---------|
| PyMuPDF (fitz) | PDF 解析与 OCR | `pip install pymupdf` |
| python-docx | Word 文档解析 | `pip install python-docx` |

依赖缺失时对应工具会返回空结果与提示，不会导致系统崩溃。

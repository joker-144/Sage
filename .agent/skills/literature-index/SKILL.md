# Literature Index Skill

文献索引与引用管理技能包 — Sage 系统学术写作的核心引擎。

## 1. 角色定位

本技能是 Sage 系统"理解文献库"的核心机制。它让 Agent 不再依赖关键词匹配，而是通过语义检索理解用户研究问题，并从已索引的论文库中精准召回相关片段。同时承担引用管理与查重验证职责，是多重验证机制中的"文献库验证"层。

## 2. 核心能力

| 能力 | 工具 | 输出 |
|------|------|------|
| 建立索引 | `index_papers` | 索引统计 {files, chunks, skipped} |
| 语义检索 | `search_literature` | 相关文献片段列表（含相似度分） |
| 引用提取 | `extract_references` | 参考文献列表（结构化） |
| 引用插入 | `insert_citation` | 标记位置替换为引用文本 |
| 格式化 | `format_references` | 目标期刊格式的参考文献列表 |
| 查重 | `check_plagiarism` | 重复率 + 重复段落详情 |

## 3. 详细执行手册

### 3.1 索引建立流程（index_papers）

#### 何时建立索引
- **必建场景**：用户上传新论文后、首次使用 search_literature 前、工作空间切换后
- **增量更新**：仅当文件 hash 变化时重新索引（默认 `force=False`）
- **强制重建**：用户明确要求或检索质量明显下降时使用 `force=True`

#### 索引前检查清单
- [ ] 已调用 `paper-processing` 技能完成文档解析（PDF/DOCX 等二进制格式）
- [ ] 工作空间内文档数量在合理范围（< 1000 篇，超出建议分库）
- [ ] 本地 embedding 模型已就绪（首次使用自动下载 all-MiniLM-L6-v2）

#### 索引后验证
- [ ] 返回的 `files` 数量与实际文档数量一致
- [ ] `chunks` 数量在合理范围（每篇论文通常 5-30 块）
- [ ] `skipped` 数量合理（过多说明文件格式异常）

### 3.2 语义检索规范（search_literature）

#### 查询构造原则
- **用研究问题而非关键词**：✓ "transformer 在长文本任务上的注意力机制改进"
                                ✗ "transformer attention"
- **包含上下文**：✓ "针对小样本学习的元学习方法与对比实验"
                  ✗ "meta-learning few-shot"
- **指定期刊偏好**：可在查询中加入"SCI一区/CSSCI"等偏好词

#### top_k 选择策略
| 场景 | 推荐值 | 理由 |
|------|--------|------|
| 快速预览 | 3 | 节省token，快速判断相关性 |
| 文献综述 | 8-10 | 全面覆盖研究脉络 |
| 引用查找 | 5 | 平衡精度与召回 |
| 查重验证 | 15 | 提高查全率 |

#### 结果使用规范
- 必须返回 `file_path` 与相似度 `score`，便于追溯
- 相似度 < 0.3 的结果应标记为"弱相关"，谨慎引用
- 引用前必须通过 `extract_metadata` 获取完整文献信息，避免引用残缺

### 3.3 参考文献提取（extract_references）

#### 支持格式与提取规则
- **Markdown (.md)**：识别 `[1] Author, *Title*, ...` 或 `(Author, 2023)` 格式
- **LaTeX (.tex)**：解析 `\bibitem{...}` 与 `thebibliography` 环境
- **Word (.docx)**：按段落分隔，识别编号格式
- **PDF (.pdf)**：从末尾 References/Bibliography 部分提取

#### 提取后结构化
每条参考文献应解析为：
```
{
  "raw": "原始文本",
  "authors": ["作者1", "作者2"],
  "year": "2023",
  "title": "论文标题",
  "venue": "期刊/会议名",
  "doi": "10.xxxx/xxxx"  // 若存在
}
```

#### 验证清单
- [ ] 作者格式统一（姓在前/名在前）
- [ ] 年份完整（4位数）
- [ ] 标题大小写正确
- [ ] DOI 格式合法（若存在）

### 3.4 引用插入流程（insert_citation）

#### 标记格式约定
撰写论文时使用占位符标记未来引用位置：
```
[CITE: transformer-original-paper]
[CITE: meta-learning-survey-2023]
```

#### 插入流程
1. 定位文件中所有 `[CITE: marker]` 标记
2. 根据 marker 关键词调用 `search_literature` 查找匹配文献
3. 调用 `extract_metadata` 获取完整文献信息
4. 按目标引用格式生成引用文本（如 `(Vaswani et al., 2017)`）
5. 调用 `insert_citation` 替换标记

#### 引用风格选择
| 引用风格 | 适用学科 | 示例 |
|---------|---------|------|
| APA | 心理学/教育学/社会科学 | (Smith, 2023) |
| MLA | 人文学科/文学 | (Smith 45) |
| GB-T7714 | 中文期刊 | [1] SMITH J. 2023... |
| Vancouver | 医学/生物 | [1] Smith J. Title... |
| Chicago | 历史/艺术 | Smith, John. *Title*... |
| IEEE | 工程/计算机 | [1] J. Smith, "Title"... |

### 3.5 格式化参考文献（format_references）

#### 输入要求
- 每行一条参考文献（原始或半结构化文本）
- 必须指定 `style` 参数，默认 APA

#### 输出规范
- 严格按目标格式输出
- 作者列表按字母序或引用顺序排列（依格式规范）
- 中文文献按 GB-T7714，英文按目标期刊要求
- 混合中英文时分别列出，中文在前

### 3.6 查重检测（check_plagiarism）

#### 调用时机
- **必查**：论文定稿前
- **建议查**：每个章节撰写完成后
- **抽查**：审校核查员执行多重验证时

#### 阈值设定
| 阈值 | 含义 | 处理建议 |
|------|------|---------|
| > 0.9 | 几乎完全重复 | 必须改写 |
| 0.8-0.9 | 高度相似 | 改写并标注引用 |
| 0.6-0.8 | 中度相似 | 适当改写+引用 |
| < 0.6 | 可接受 | 标注引用即可 |

#### 输出解读
- `total_similarity`: 整体重复率
- `matched_segments`: 重复段落详情（含来源文件路径）
- 重复段落需逐条分析，区分"合理引用"与"不当重复"

## 4. 与其他技能的协作

| 协作技能 | 协作方向 |
|---------|---------|
| `paper-processing` | 上游：提供已解析的文档文本供索引 |
| `external-search` | 并行：本地索引未命中时调用外部检索补充 |
| `writing-assistant` | 下游：检索结果用于支撑写作 |
| `ai-pattern-reducer` | 后置：改写后重新查重验证 |

## 5. 多重验证机制中的角色

本技能在 Sage 多重验证机制中承担**文献库验证层**：
1. **引用真实性验证**：所有引用必须能在本地文献库或外部检索中找到对应文献
2. **作者年份核验**：引用的作者、年份与原文档元数据匹配
3. **DOI 验证**：有 DOI 的引用通过 `external-search` 中的 `search_crossref` 验证合法性
4. **查重验证**：定稿前必须通过 `check_plagiarism` 检测

## 6. 质量标准

- 索引覆盖率：工作空间所有可索引文档 100%
- 语义检索 Top-5 召回率 ≥ 85%（人工评估）
- 参考文献提取准确率 ≥ 90%
- 引用插入位置准确率 100%
- 格式化输出合规率 100%
- 查重检测误报率 < 5%

## 7. 智能体自主判断事项

以下决策由智能体根据上下文自主判断，本技能包不强制规定：
- 是否需要强制重建索引（force 参数）
- top_k 取值（基于查询意图）
- 引用风格选择（基于目标期刊）
- 查重阈值（基于论文类型）
- 是否调用外部检索补充本地文献库不足
- 多重验证触发时机（写作过程中 vs 定稿前）

## 8. 依赖说明

| 依赖 | 用途 | 自动加载 |
|------|------|---------|
| sentence-transformers | 本地 embedding | 首次使用自动下载 |
| numpy | 向量计算 | 已在核心依赖 |
| SQLite | 向量与元数据存储 | Python 标准库 |

## 9. 注意事项

- 首次索引会下载 embedding 模型（约 80MB），建议设置 `HF_ENDPOINT=https://hf-mirror.com`
- 索引数据库与工作空间绑定，切换工作空间需重新索引
- 查重仅对比本地文献库，外部文献查重需配合 `external-search` 技能
- 引用插入是文件写入操作，会修改原文件，建议先备份

# AI Pattern Reducer Skill

降AI味技能包 — Sage 系统论文真实性的"最后一道防线"。

## 1. 角色定位

本技能是 Sage 系统应对"AI检测"与"学术诚信"双重压力的关键能力。由于 Sage 本身是AI辅助写作系统，产出的初稿不可避免带有AI生成痕迹（机械句式、过度连接、空洞修饰）。本技能在论文定稿前执行，将AI痕迹降至人类学者写作水平，同时保留论证内容与引用完整性。

**核心原则**：改写表达，不改原意；保留引用，重塑句式；遵守学术诚信，不虚构内容。

## 2. 核心能力

| 能力 | 工具 | 输出 |
|------|------|------|
| AI模式识别 + 改写 | `reduce_ai_pattern` | 改写后文本 + 改写说明 + AI特征评估 |

## 3. 详细执行手册

### 3.1 AI写作模式识别清单

#### 3.1.1 句式层面的AI特征
| 特征 | 典型表现 | 人类学者对比 |
|------|---------|-------------|
| 句式单一 | 多句以"然而"/"此外"/"因此"开头 | 句式多样，长短交错 |
| 平行结构过度 | "不仅...而且...更..."频繁出现 | 偶尔使用，自然变化 |
| 主谓宾机械 | 每句都是"主语+动词+宾语"结构 | 倒装/省略/复杂句式穿插 |
| 长句堆砌 | 单句超过40字，多重从句 | 长短结合，重点突出 |

#### 3.1.2 词汇层面的AI特征
| 特征 | 典型表现 | 替换建议 |
|------|---------|---------|
| 空洞修饰 | "显著""重要""关键"过度使用 | 具体数据/程度副词 |
| 套话连接 | "综上所述""总而言之""由此可见" | 自然过渡或直接陈述 |
| 学术腔过浓 | "本研究旨在""具有重要意义" | 直接陈述事实 |
| 重复用词 | 同一段多次使用相同形容词 | 同义词替换 |

#### 3.1.3 结构层面的AI特征
| 特征 | 典型表现 | 改写策略 |
|------|---------|---------|
| 三段式机械 | 每段都是"主题句+论据+总结" | 打破固定结构 |
| 罗列式论证 | "首先...其次...最后..." | 逻辑递进式表达 |
| 平衡式对比 | "一方面...另一方面..." | 不对称论述 |
| 总结性结尾 | 每段都有"总之..." | 自然收尾 |

#### 3.1.4 语义层面的AI特征
| 特征 | 典型表现 | 改写方向 |
|------|---------|---------|
| 过度概括 | "研究表明..."不指明具体研究 | 补充具体引用 |
| 自我指涉 | "如前所述""上文提到" | 直接复述关键点 |
| 假设性表述 | "可能""或许""大概"过度 | 明确判断或保留必要的不确定性 |
| 元话语过多 | "本文将讨论""接下来分析" | 直接展开内容 |

### 3.2 改写流程（reduce_ai_pattern）

#### Step 1：识别AI特征
扫描全文，标记所有AI特征点：
```
[AI-句式] 段落3第2句：以"此外"开头的机械连接
[AI-词汇] 段落5第1句："显著"为空洞修饰
[AI-结构] 段落7：典型三段式结构
[AI-语义] 段落9第3句：过度概括，无具体引用
```

#### Step 2：评估AI特征密度
```
AI特征密度 = AI特征点数量 / 总句数

密度 < 10%：轻微，仅需局部改写
密度 10-25%：中度，需要系统性改写
密度 > 25%：重度，建议重构段落
```

#### Step 3：按优先级改写
改写优先级（高→低）：
1. **严重AI特征**：句式完全雷同、明显套话 → 必须改写
2. **中度AI特征**：空洞修饰、机械连接 → 建议改写
3. **轻微AI特征**：偶尔重复用词 → 可不改

#### Step 4：改写策略
| 策略 | 操作 | 示例 |
|------|------|------|
| 句式变换 | 主动↔被动/陈述↔倒装 | "研究表明X" → "X的研究显示" |
| 词汇替换 | 同义词/具体化 | "显著" → "提升23.5%" |
| 结构重组 | 拆分/合并段落 | 长段拆为2段，重点突出 |
| 过渡改写 | 自然衔接 | "此外" → 直接陈述新观点 |
| 引用补充 | 添加具体引用 | "研究表明" → "(Smith, 2023)" |
| 冗余删除 | 删除无信息量句子 | 删"具有重要意义" |

#### Step 5：保留要素清单
改写时必须保留：
- [ ] 所有原始引用 `[CITE: xxx]` 占位
- [ ] 核心论点与论证逻辑
- [ ] 关键数据与统计结果
- [ ] 专业术语（除非作者使用不一致）
- [ ] 章节结构（不调整章节顺序）

#### Step 6：输出格式
```
=== 改写结果 ===
[改写后的完整文本]

=== AI特征评估 ===
原始AI特征密度：[百分比]
改写后AI特征密度：[百分比]
降低幅度：[百分比]

=== 主要改写说明 ===
1. [段落X] 句式变换：[原文] → [改写]
   原因：[机械连接，改为自然过渡]
   
2. [段落Y] 词汇替换：[原文] → [改写]
   原因：[空洞修饰，改为具体数据]

3. [段落Z] 结构重组：[操作描述]
   原因：[三段式机械，打破固定结构]

=== 保留要素核对 ===
- [✓] 引用占位完整保留：[数量]个
- [✓] 核心论点未改变
- [✓] 数据准确性未受影响
- [✓] 专业术语保留
```

### 3.3 学科差异化处理

#### 理工科 (SCI/EI)
- **容忍度高**：被动语态、客观陈述是规范
- **重点改写**：句式单一、套话连接
- **保留**：数据描述、方法步骤的程式化表达

#### 人文社科 (SSCI/CSSCI)
- **容忍度低**：要求论证性与文采
- **重点改写**：机械结构、空洞修饰、过度概括
- **保留**：理论对话、批判性论述

#### 医学 (Vancouver)
- **容忍度高**：流程化表达是规范
- **重点改写**：仅改套话与重复
- **保留**：方法描述的标准化结构

### 3.4 与查重的协同

降AI味与查重（`check_plagiarism`）存在协同关系：

```
查重发现重复段落
    ↓
判断重复原因：
    - AI生成内容雷同 → 调用 reduce_ai_pattern 改写
    - 引用未规范化 → 调用 insert_citation 添加引用
    - 真实抄袭 → 必须删除重写
    ↓
改写后重新查重
    ↓
循环直至通过
```

## 4. 多智能体协作中的角色

| 协作阶段 | 调用方 | 调用时机 |
|---------|--------|---------|

## 6. 质量标准

- AI特征密度降低率 ≥ 60%（改写后 vs 改写前）
- 原意保留率 100%（不改变论证内容）
- 引用占位保留率 100%
- 数据准确性 100%（不修改任何数值）
- 专业术语保留率 100%
- 改写后通过查重检测（重复率 < 阈值）
- 改写后通过逻辑检查（`check_logic` 不引入新问题）

## 7. 智能体自主判断事项

以下决策由智能体根据上下文自主判断：
- 改写力度（轻度/中度/重度）
- 是否触发改写（基于AI特征密度评估）
- 改写优先级（基于严重程度）
- 学科差异化处理策略
- 是否需要循环改写（改写后再次评估）
- 是否与查重协同（基于查重结果）
- 改写范围（全文/特定章节/特定段落）

## 8. 注意事项

- **学术诚信底线**：改写表达，不虚构内容、不编造数据、不伪造引用
- **不可改写内容**：数据、统计结果、引用、专业术语、章节标题
- **改写不可过度**：保持学术严谨性，不追求"完全不像AI"而损失专业性
- **学科差异**：理工科容忍度高于人文社科，避免一刀切
- **循环验证**：改写后必须重新查重 + 逻辑检查，确保未引入新问题
- **用户确认**：重度改写建议用户确认后再采纳
- **保留改写记录**：便于追溯与回滚

## 9. 禁止行为

- ❌ 修改任何数据与统计结果
- ❌ 删除或修改原始引用占位
- ❌ 改变核心论点与论证逻辑
- ❌ 替换专业术语（除非作者使用不一致）
- ❌ 调整章节顺序
- ❌ 添加未经验证的新论据
- ❌ 编造引用以"显得更学术"

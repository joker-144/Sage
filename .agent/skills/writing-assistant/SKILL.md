# Writing Assistant Skill

学术论文写作辅助技能包 — Sage 系统的写作质量保障核心。

## 1. 角色定位

本技能是 Sage 系统中"产出学术内容"的核心能力。不同于通用 LLM 写作，本技能强制约束输出符合 SCI/SSCI/CSSCI/EI 等高水平期刊的学术规范：论证结构严谨、术语统一、引用规范、逻辑闭环。撰写员（Coder Agent）在协作讨论中调用本技能产出初稿，审校核查员（Reviewer Agent）调用 `check_logic` 验证论证完整性。

## 2. 核心能力

| 能力 | 工具 | 输出 |
|------|------|------|
| 大纲生成 | `generate_outline` | 结构化论文大纲 |
| 段落撰写 | `write_paragraph` | 学术化段落文本（含引用占位） |
| 学术润色 | `polish_academic` | 润色后文本 + 修改说明 |
| 逻辑检查 | `check_logic` | 逻辑问题列表 + 修改建议 |

## 3. 详细执行手册

### 3.1 大纲生成规范（generate_outline）

#### 大纲类型选择
| 类型 | 适用论文 | 结构特征 |
|------|---------|---------|
| IMRAD | 实证研究（SCI/EI） | Introduction/Methods/Results/Discussion |
| 主题式 | 综述/理论（SSCI/CSSCI） | 按主题分章节递进 |
| 综述式 | 文献综述 | 时间线/方法论/学科脉络 |
| 案例式 | 案例研究 | 案例背景/分析/讨论/启示 |

#### 输出结构
```markdown
# 论文标题

## 1. 引言 (Introduction)
### 1.1 研究背景
### 1.2 研究问题
### 1.3 研究意义与贡献

## 2. 文献综述 (Literature Review)
### 2.1 理论基础
### 2.2 研究现状
### 2.3 研究空白

## 3. 研究方法 (Methodology)
### 3.1 研究设计
### 3.2 数据与样本
### 3.3 分析方法

## 4. 结果 (Results)
### 4.1 描述性统计
### 4.2 假设检验

## 5. 讨论 (Discussion)
### 5.1 主要发现
### 5.2 理论贡献
### 5.3 实践启示
### 5.4 局限与未来研究

## 6. 结论 (Conclusion)

## 参考文献 (References)
```

#### 大纲生成检查清单
- [ ] 每个章节有明确的研究问题对应
- [ ] 章节间逻辑递进（不重复不跳跃）
- [ ] 包含"研究空白"识别（文献综述必备）
- [ ] 包含"研究贡献"声明（引言/讨论必备）
- [ ] 包含"研究局限"自省（讨论必备）
- [ ] 引用占位标记 `[CITE: xxx]` 已预置

### 3.2 段落撰写规范（write_paragraph）

#### 段落结构原则（PEEL/MEAL）
- **P/M (Point/Main idea)**：主题句明确陈述论点
- **E (Evidence)**：提供证据（数据/引用/案例）
- **E (Explanation)**：解释证据与论点关系
- **L (Link)**：承接上下文，呼应研究问题

#### 撰写流程
1. **明确段落功能**：是论证/描述/分析/对比？
2. **调用 `search_literature`** 检索相关文献支撑
3. **撰写主题句**：明确段落核心论点
4. **展开论证**：每条论据配引用占位 `[CITE: xxx]`
5. **过渡衔接**：与前段/后段逻辑连接
6. **自检**：调用 `check_logic` 验证段落内部逻辑

#### 学科风格适配
| 学科 | 风格特征 | 注意事项 |
|------|---------|---------|
| 理工 (SCI/EI) | 客观、被动语态、数据驱动 | 避免主观判断，强调可重复性 |
| 人文 (SSCI) | 论证性、引文密集、理论对话 | 平衡多方观点，体现批判性 |
| 中文期刊 (CSSCI) | 规范严谨、政策语境 | 注意中国学者表述习惯 |
| 医学 (Vancouver) | 严谨、流程化 | 遵循 CONSORT/PRISMA 规范 |

#### 引用占位规范
撰写时使用 `[CITE: 描述性关键词]` 标记未来引用位置：
```
近年来，深度学习在医学影像识别上取得显著进展 [CITE: medical-imaging-deep-learning-survey]。
```
后续由 `citation` Agent 调用 `insert_citation` 替换为真实引用。

### 3.3 学术润色规范（polish_academic）

#### 润色维度
1. **术语统一**：全篇同一概念用同一术语
2. **句式优化**：长句拆分、主动被动转换、避免口语化
3. **学术化转换**：
   - "我觉得" → "本研究认为" / "研究表明"
   - "很好" → "显著" / "具有显著优势"
   - "因为...所以" → "由于...因此" / "基于...导致"
4. **冗余消除**：删除无信息量的修饰词
5. **时态规范**：方法部分用过去时，讨论部分可现在时

#### 输出格式
```
=== 润色结果 ===
[润色后的文本]

=== 修改说明 ===
1. [修改点1] 原因：[理由]
2. [修改点2] 原因：[理由]
...

=== 学术化转换对照 ===
| 原文 | 润色后 | 类型 |
|------|--------|------|
| 我觉得 | 研究表明 | 主观化 |
```

#### 润色禁忌
- **不得改变原意**：仅优化表达，不修改论证
- **不得添加论据**：保持作者原意
- **不得删除引用**：保留所有原始引用占位
- **不得替换术语**：除非作者使用不一致

### 3.4 逻辑检查规范（check_logic）

#### 检查维度
1. **因果链完整性**：A→B→C 是否每步都有证据支撑
2. **证据充分性**：论点是否有足够文献/数据支撑
3. **逻辑跳跃识别**：是否存在未论证的假设
4. **循环论证**：是否用结论反推前提
5. **过度概括**：从有限样本推出普适结论
6. **矛盾检测**：前后论述是否自相矛盾
7. **替代解释排除**：是否考虑竞争性假设

#### 输出格式
```
=== 逻辑检查报告 ===

总体评价：[通过/需修改/需重大修改]

发现的问题：
1. [严重] [位置] 因果链断裂
   描述：从X直接跳到Z，缺少Y的论证
   建议：补充Y的论证或文献支撑
   
2. [中等] [位置] 证据不足
   描述：论点仅基于单一文献
   建议：补充2-3篇相关文献交叉验证
   
3. [轻微] [位置] 表述模糊
   描述："研究表明"未指明具体研究
   建议：明确标注 (作者, 年份)
```

#### 严重等级标准
| 等级 | 含义 | 处理建议 |
|------|------|---------|
| 严重 | 逻辑断裂/循环论证/重大矛盾 | 必须修改，影响论文成立 |
| 中等 | 证据不足/过度概括 | 建议修改，影响说服力 |
| 轻微 | 表述模糊/术语不一致 | 优化即可，不影响核心论证 |

## 4. 多智能体协作中的角色

本技能在 Sage 多智能体协作中担任核心生产能力：

| 协作阶段 | 调用方 | 调用工具 |
|---------|--------|---------|
| 方法论设计 | Planner Agent | `generate_outline` |
| 平等协作讨论 | Coder Agent | `write_paragraph` + `search_literature` |
| 整合后润色 | Consolidator Agent | `polish_academic` |
| 审校验证 | Reviewer Agent | `check_logic` |
| 修订迭代 | Debugger Agent | `write_paragraph`（修订） |

## 5. 与其他技能的协作

| 协作技能 | 协作方式 |
|---------|---------|
| `literature-index` | 撰写前调用 `search_literature` 检索支撑文献 |
| `external-search` | 本地文献不足时补充外部检索 |
| `ai-pattern-reducer` | 润色后调用降低AI写作痕迹 |
| `paper-processing` | 引用文献时获取元数据 |

## 6. 质量标准

- 大纲章节完整性 100%（含引言/方法/结果/讨论/结论）
- 段落结构规范率 ≥ 95%（PEEL/MEAL 结构）
- 学术化转换准确率 ≥ 90%
- 逻辑问题检出率 ≥ 85%（人工复核）
- 引用占位预置率 100%（每个论据对应一个占位）

## 7. 智能体自主判断事项

以下决策由智能体根据上下文自主判断：
- 大纲类型选择（IMRAD/主题式/综述式）
- 段落撰写顺序（按大纲/按优先级）
- 润色力度（轻度/中度/重度）
- 逻辑检查触发时机（每段/每章/定稿）
- 引用占位密度（保守/适中/密集）
- 学科风格适配（基于目标期刊）

## 8. 注意事项

- 撰写过程中所有引用必须用 `[CITE: xxx]` 占位，不直接编造引用
- 润色不改变原意，仅优化表达
- 逻辑检查报告需逐条响应，不可忽略严重问题
- 多智能体协作产出需经 `polish_academic` 统一文风后整合
- 修订时优先处理严重逻辑问题，再处理中轻微问题
